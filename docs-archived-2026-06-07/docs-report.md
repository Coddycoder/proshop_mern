## Undersize chunks (< 150 tokens) — 132

Most are intentional per spec §2 ("hard floor: a semantically complete tiny
section may be left small if merging hurts coherence"). Top contributors:

- **Runbooks (~48)**: procedural files decompose into many short H3 atomic
  steps (one code block per step). Merging across H3 boundaries would
  destroy step-by-step semantics.
- **Glossary (~12)**: short term definitions like *Wishlist*, *Async Handler*,
  *P95 Latency* are each ~50–100 tokens; merging would conflate unrelated terms.
- **Preambles (~10)**: file-level intro paragraphs before the first `H2`
  (kept separate; cross-file merge is not permitted).
- **Best-practice subsections (~6)**: e.g. *2.3 Fastify*, *3.6 Soft deletes* —
  each is a complete recommendation; merging across topics hurts retrieval.
- **One ADR chunk**: `adrs_adr-002-redux-vs-context__005` (128 t, *Current
  Assessment*) — could not merge backward (would exceed 600) and has no
  forward sibling.

The number is large but consistent with the corpus structure
(many small atomic sections in glossary/runbooks). They are still
embeddable; the cost is mostly index entries, not retrieval quality.

## Caveats and follow-ups

1. **Token estimator is `chars/4`**, not the real embedding model's
   tokenizer. The 600/150 thresholds are therefore approximate. If the
   downstream model is `text-embedding-3-small` or similar, real token
   counts may run ±15%. Not material for retrieval; flag if migrating to a
   model with a tight context window.
2. **Preamble chunks include the H1** in both the breadcrumb prefix and
   the body content. This is per spec §4 ("original headings preserved
   below breadcrumb") but produces a visible doubled title in those 6–8
   chunks. Acceptable; trivial to strip downstream if needed.
3. **The 1566-token table** in `features-analysis-ru.md` is the only true
   outlier. If you want to support questions like "what is the status of
   flag X?", consider also indexing `features.json` as a separate pass
   (1 chunk per flag) — see brainstorm history.
